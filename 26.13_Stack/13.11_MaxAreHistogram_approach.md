# Approach for `maxArea2` in `13.11_MaxAreHistogram.java`

## Objective
Compute the maximum rectangular area under a histogram in linear time using the concept of next smaller elements.

## Strategy
1. Use two arrays to store the index of the next smaller element on the right (`nsr`) and the next smaller element on the left (`nsl`) for each bar.
2. For each bar at index `i`, the rectangle can extend from `nsl[i] + 1` to `nsr[i] - 1` while maintaining the bar height `arr[i]`.
3. The width of the maximal rectangle for bar `i` is `nsr[i] - nsl[i] - 1`.
4. The area for bar `i` is `arr[i] * width`.
5. The answer is the maximum area found across all bars.

## Implementation details
- Use a stack of indices to compute the next smaller element positions.
- For `nsr` (next smaller to right): iterate from right to left. Pop stack indices while current bar is smaller or equal to the bar at the top of the stack.
  - If stack becomes empty, set `nsr[i] = arr.length`.
  - Otherwise, set `nsr[i] = st.peek()`.
- For `nsl` (next smaller to left): iterate from left to right. Pop while current bar is smaller or equal.
  - If stack becomes empty, set `nsl[i] = -1`.
  - Otherwise, set `nsl[i] = st.peek()`.
- Finally compute the max area using the precomputed `nsl` and `nsr`.

## Complexity
- Time: O(n)
- Space: O(n)

## Notes
- Using `>=` in the stack comparisons ensures equal-height bars are handled correctly and avoids over-counting width.
- The algorithm is equivalent to computing the largest rectangle in a histogram using boundary positions for each bar.
